# Installation

`@noy-db/ui-nuxt` is the **Nuxt binding** of the noy-db UI family — it adds the components, the `core/`
config layer, and a Nuxt module on top of the agnostic `@noy-db/ui` base. For the concepts it builds on
(schema-driven UI, the search engine, design tokens), see the **@noy-db/ui** docs.

```bash
npm i @noy-db/ui-nuxt
```

It's a **Nuxt module** that ships **pre-compiled CSS** — your app needs no UnoCSS/Tailwind of its own
to consume it. The base `@noy-db/ui` is installed automatically as its dependency.

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  modules: ['@noy-db/ui-nuxt/module'],
  noydbUi: { theme: 'system', locale: 'en' },   // configure once
})
```

The module injects the tokens + utilities CSS before your own, so your `--nui-*` brand map wins.

### Brand it

The whole library is driven by `--nui-*` CSS variables (shipped by `@noy-db/ui`). Map them to your
brand on `:root`:

```css
:root {
  --nui-accent: #e11d48;   /* your brand */
  --nui-accent-fg: #fff;
  /* --nui-fg / --nui-muted / --nui-border / --nui-bg / --nui-bg-accent … */
}
```

Full token reference: **@noy-db/ui › Design tokens**. Dark mode ships built-in — see [Theme](./4.theme.md).

### Peer requirements

`vue` ^3.5 and `@noy-db/hub` ^0.2 (the vault runtime).
