# Components

The **top shelf** — the components you compose. (Internal sub-components like the header filters and
the popover primitive are in `internal/`; you never place them directly.)

## List family — collections

| Component | Main function | Key features |
|---|---|---|
| **CollectionList** | render a collection as a list/table | sticky subtotals, responsive columns + xs-stacking, grouped rendering, cell slots, multi-sort, `anchorKey` scrolls + flash-highlights a row once (e.g. returning from a detail's `back()`) |
| **SearchBox** | pills query box (`v-model` = query string) | field/value autocomplete, two-step disambiguation, keyboard nav |
| **SavedSearchMenu** | saved searches | favorite, rename, delete, set-default, "rolling" badge |
| **RecentSearchMenu** | recent queries | applied as pills + relative time |
| **NlSearchButton** | natural-language search (presentational) | BYO key entry, refine mode, emits intents |
| **ColumnChooser** | per-user column visibility | auto → show → hide cycle, persisted |
| **GroupByControl** | group-by chain | nested groups, expand/collapse to level |

## Item family — records

| Component | Main function | Key features |
|---|---|---|
| **RecordDetail** | a record's read view | fields → responsive cards derived from `describe()`'s `group`/`order` (the `groups` prop is now just an override), container-measured grid + density tiers (shared 448/640/992 ladder), labels/units/enum-labels from `describe()` — read-mode label precedence: the hub-dressed `<field>Label` sibling (a `{ locale }` read) › the `options` prop (active app locale) › the dict's declared-locale label › the raw code, i18n fields read with `{ locale: 'raw' }` render one badged row per language, PII masked via `sensitivity`, entity fields as links (transversal nav via `routeFor` + `navigate`; a bare `ref` without `displayFor` links too, named via `options`); **in-place edit mode** — `editing`/`draft` (paired with `useRecordItem`) morphs editable cells into their input widgets in the same grid cells, required marks + constraint hints via `fieldHint()`, per-field `errors` from a failed `put()`, lock affordance on read-only fields |
| **RecordForm** | schema-driven create/edit | inputs from `widget`/`editable` (text/textarea/number/date/select/checkbox); host calls `put()` (noy-db zod-validates) and feeds back per-field `errors`; card-matching layout; shares its widget renderer with RecordDetail's edit mode via the internal `FieldControl` component |
| **TraverseBar** | sticky found-set stepper above a detail view | narrated-search breadcrumb doubles as "back to list"; fixed vertical cluster (⤒ first, ▲ prev, N/M, ▼ next, ⤓ last) with destination-named tooltips/labels and a tabular-nums reserved width so it never reflows; `j`/`k` and `⌥↑`/`⌥↓` step; a popover scrubber (internal `TraverseScrubber`) offers a typed jump + grouped mini-list; goes inert while `editing` |
| **ItemPath** | a detail's path-shaped title | breadcrumb trail — the group-by chain when the found set was grouped, else the entity's natural ref-axis — terminating in the record's title; collapses to first › … › terminal below 448px; group segments emit `back`, entity segments emit `navigate` |

```vue
<RecordDetail :record="buyer" :fields="vault.collection('buyers').describe().fields"
  :route-for="(c, id) => `/${c}/~${id}`" editable @edit="open" @navigate="go" />
```

`RecordForm`'s `errors` prop is `Record<fieldKey, message>`. Turn a failed `put()` into that map with
**`fieldErrors()`** (from `@noy-db/ui`) — it decomposes a `SchemaValidationError`'s issues by field;
it also maps noy-db's `MissingTranslationError` to its offending field, same shape either way; a
non-validation error yields `{}` so you fall back to a generic banner:

```ts
import { fieldErrors } from '@noy-db/ui'
try { await vault.collection('buyers').put(id, record) }
catch (e) { formErrors.value = fieldErrors(e) }   // → { vatNumber: 'Invalid VAT number', … }
```
