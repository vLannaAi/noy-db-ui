# Configuration

`@noy-db/ui-nuxt/core` is the **only** API you configure. Set it once; every component reads it, with safe
defaults if you don't.

```ts
import { provideNoydbUi } from '@noy-db/ui-nuxt/core'

provideNoydbUi({
  theme: 'system',          // 'light' | 'dark' | 'system'
  locale: 'it',
  t: (key, fallback) => i18n.t(key) ?? fallback,   // UI-string translation
  llm: anthropicClient,     // AI features (NL search, AI-fill…)
  voice: undefined,         // STT source; omit → browser Web Speech API
})
```

Nothing here is required — the library renders fully unconfigured.

- [Theme](./4.theme.md) — dark/light wiring (brand tokens live in @noy-db/ui › Design tokens).
- [AI & Voice](./5.ai-and-voice.md) — natural-language + speech-to-text.
- **i18n** — every built-in UI string routes through your `t` under the **`nui.*`** key namespace
  (e.g. `nui.clear`, `nui.save`, `nui.search.placeholder`, `nui.filter.title`), each with an English
  fallback — so it's fully usable unconfigured, and fully translatable when you supply `t`. Data labels
  come from `describe()`/`i18nText`.
- **Responsive / touch** — `useViewport()` exposes width + coarse-pointer for touch variants.
