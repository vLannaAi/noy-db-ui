# AI & Voice

Both are **pluggable layers**, injected via `core/` — the components only know the interface.

## AI / LLM

Provide an `LlmClient`; the host owns the provider + key (e.g. Anthropic BYO-key, local-first).

```ts
import { provideNoydbUi, type LlmClient } from '@noy-db/ui-nuxt/core'
const llm: LlmClient = { complete: ({ system, user }) => myProvider.complete(system, user) }
provideNoydbUi({ llm })
```

`NlSearchButton` turns plain language into a query DSL via this client. Unconfigured → AI search is
simply unavailable; everything else works.

## Voice (speech-to-text)

`useVoiceInput()` defaults to the browser **Web Speech API**; swap the source without touching
callers.

```ts
import { useVoiceInput } from '@noy-db/ui-nuxt/core'
const { listening, transcript, supported, toggle } = useVoiceInput({ onFinal: (t) => (query.value = t) })
```

**Built in.** `SearchBox` and `NlSearchButton` already use this — a mic button appears automatically
whenever a source is available (Web Speech, or one you inject). Dictating into `SearchBox` commits a
search term; dictating into `NlSearchButton` fills the natural-language request. Call `useVoiceInput()`
yourself only for your own fields.

**Swappable source** — ship Web Speech now (no key, Chrome/Safari, cloud STT); drop in an on-device
engine (Google AI Edge / LiteRT, Whisper-WASM) later for fully-local transcription, with **no UI
change**.

```ts
provideNoydbUi({ voice: myOnDeviceVoiceSource })  // implements { supported, start, stop }
```
