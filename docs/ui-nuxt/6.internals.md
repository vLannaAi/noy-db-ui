# Internals

The top-shelf components render these for you — you almost never place them directly. They're
documented for the curious and for anyone extending the binding.

- **`internal/` — sub-components.** The pieces a top-shelf component composes: `ColumnHeader`,
  `HeaderFilter{Cell,Enum,Entity,Date}`, `FilterChips`, and the accessible `Popover` primitive
  (Esc / outside-click / focus-return, ARIA). They're **explicitly imported** by the components,
  never auto-registered — so they stay out of your component namespace.

The engine these sit on top of (the search AST, `useCollectionList`, `schemaFromDescribe`, joins, table
utils) is framework-agnostic and lives in `@noy-db/ui` — see **@noy-db/ui › Search engine**. If you're
extending behaviour rather than presentation, start there.
