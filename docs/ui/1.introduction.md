# Introduction

**noy-db UI** turns a [noy-db](https://github.com/vLannaAi/noy-db) vault into screens. A noy-db
collection self-describes via `collection.describe()` — every field's label, type, unit, enum set,
entity ref and edit widget. The family turns that metadata into the UI you'd otherwise rebuild for
every entity: **list, detail, search, edit, and AI/voice query** — with no per-collection code.

It is **domain-free** (it knows nothing about your business), **design-system-free** (it forces no
look on your app — everything is `--nui-*` CSS variables), and **local-first** (AI keys and speech
stay on device).

## The packages

The family follows one rule — **bare name = the agnostic base, `<framework>` suffix = the binding**
(mirroring noy-db's `@noy-db/in-*` integrations):

| Package | What it is | Docs |
|---|---|---|
| **`@noy-db/ui`** | the framework-agnostic **base** — search/list engine, schema model, design tokens. Pure TS/CSS. | *(here)* |
| **`@noy-db/ui-nuxt`** | the **Nuxt binding** — components + module + the `core/` config layer | its own docs |
| *(future)* `@noy-db/ui-next`, `@noy-db/ui-react` | other bindings, same base | — |

This base renders nothing itself; a binding adds the components on top. These docs cover the parts
**every binding shares**:

- **[Schema-driven](./2.schema-driven.md)** — how `describe()` powers columns, filters, widgets, joins.
- **[Search engine](./3.search-engine.md)** — the query AST/DSL pipeline + `useCollectionList`.
- **[Design tokens](./4.design-tokens.md)** — the `--nui-*` variables you reskin with.
- **[Architecture](./5.architecture.md)** — the two-axis / three-tier organisation of the family.

## The component families

A binding's components map 1:1 to noy-db's domains, so the set is principled, not arbitrary:

| Family | noy-db domain | Purpose |
|---|---|---|
| **List** | collections | browse / search / filter / group / sort |
| **Item** | records | read detail + schema-driven edit |
| **Asset** *(soon)* | blobs | view / gallery |
| **Auth** *(soon)* | keyring / sessions | unlock / manage keys |
| **Owner** *(soon)* | identity & federation | identity & cross-vault access |

→ To install and use them, see your binding's docs (e.g. **@noy-db/ui-nuxt › Installation**).
