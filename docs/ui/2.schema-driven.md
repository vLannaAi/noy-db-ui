# Schema-driven

Everything visual derives from `collection.describe()`:

| `describe()` gives | drives |
|---|---|
| `label` | column header, form label |
| `semanticType` (date/currency/percent/country/vat/entity…) | filter kind, alignment, number format |
| `dict` (enum values + labels) | faceted enum filter, badges |
| `ref` / `displayFor` | entity link + transversal navigation |
| `unit`, `sensitivity`, `widget`, `aggregate` | unit display, PII masking, edit input, subtotals |

```ts
import { schemaFromDescribe, joinedSchema, joinedRows } from '@noy-db/ui'

// one collection
const schema = schemaFromDescribe('sales', col.describe().fields)

// or joined collections (sales ⋈ buyers) — same engine
const schema2 = joinedSchema(salesSchema, [{ schema: buyersSchema, rows: buyers, localKey: 'buyerId' }])
```

The handful of facts `describe()` can't carry (column order, responsive `relevance`, `stackWith`) stay
**view-side** — passed as overrides. See [Architecture](./5.architecture.md) and the
[Search engine](./3.search-engine.md).
