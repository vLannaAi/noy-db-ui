# Search engine

The engine is the heart of `@noy-db/ui` — a framework-agnostic pipeline that turns a query **string**
into filtered, sorted, grouped rows plus the facets and pills a UI renders. It has no knobs and no
domain; you feed it a schema (from `describe()`) and rows.

## The pipeline

```
query string → tokenize → parse → resolve → evaluate → group / summary / pills
```

| Stage | Module | Does |
|---|---|---|
| tokenize | `tokenize` | split the raw string into tokens |
| parse | `parse` | tokens → a search **AST** (`ast`) |
| resolve | `resolve` | bind field names + values against the schema (synonyms via `registerSynonyms`) |
| evaluate | `evaluate` | apply the AST as filter + sort over rows |
| group / summary | `group`, `summary` | grouping chains, subtotals, enum breakdowns |
| pills | `pills` | the AST rendered back as removable filter/sort/group chips |
| dates | `dates` | rolling-date tokens (`thisYear`, `last30d`, …) |
| suggest | `suggest` | field/value autocompletion |

```ts
import { parse, resolve, evaluate, serialize } from '@noy-db/ui'

const ast = resolve(parse('status:paid total>1000 sort:-saleDate'), schema)
const rows = evaluate(ast, allRows)
const round = serialize(ast)   // back to a canonical query string
```

## `useCollectionList`

The composable that wires the whole pipeline to reactive state — query string ⇄ AST → rows / facets /
subtotals / grouping / column prefs. (Uses Vue's reactivity, so the base lists `vue` as a peer.)

```ts
import { useCollectionList } from '@noy-db/ui'

const list = useCollectionList({ schema, rows, query })
//   → list.rows, list.facets, list.subtotals, list.groups, …
```

## Schema + joins

`schemaFromDescribe` builds the engine's schema from a collection's `describe()`; `joinedSchema` /
`joinedRows` extend the **same** engine over several collections without any special-casing — see
[Schema-driven](./2.schema-driven.md).

## Saved / recent / NL

`saved`, `history`, and `nl` provide saved searches, recent-query history, and the natural-language →
DSL prompt helpers. They're domain-free; the host supplies storage and (for NL) an LLM client — the
binding wires those in via its config layer.

> Extending the engine? Start with `useCollectionList` and the module READMEs in `src/`.
