# Design tokens

`@noy-db/ui` ships the family's design tokens as a single stylesheet — `@noy-db/ui/tokens.css`. Every
binding's components read these `--nui-*` variables (resolved through `color-mix`), so the **whole
library reskins by setting them on `:root`** — no rebuild, no fork.

```ts
import '@noy-db/ui/tokens.css'
```

## The variables

```css
:root {
  --nui-accent: #e11d48; --nui-accent-fg: #fff;   /* brand */
  --nui-fg: #0f172a; --nui-muted: #64748b; --nui-subtle: #94a3b8;
  --nui-border: #e2e8f0; --nui-bg: #fff; --nui-bg-accent: #f1f5f9;
  --nui-danger: #dc2626;
}
```

Override any subset on `:root` to brand the library; the defaults are a neutral slate so it looks
finished out of the box.

## Dark mode

`tokens.css` ships a dark token set behind `data-theme="dark"` on `<html>`. The variables swap; the
components carry **zero theme logic**. Setting the attribute is the binding's job — e.g. the Nuxt
binding's `useTheme()` (see **@noy-db/ui-nuxt › Theme**).

## Namespacing

Tokens are namespaced `--nui-*` (and the utility classes `nui-*`) on purpose: a host app on Tailwind
or another design system can consume the library with **no collision** — its own variables and the
library's never overlap.
